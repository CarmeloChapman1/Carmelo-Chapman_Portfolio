Rotation II: Data Management

Learning how enterprise data environments are structured, maintained, and used to support the business.

Team & Focus

In the Data Management rotation, I worked alongside a team responsible for handling large-scale data workflows. Even though I did not own my own user story, I was able to learn how different technologies and environments fit together in an enterprise data platform.

Key Activities

  • Completed onboarding for SQL DataStage workflows and related processes.
  • Was introduced to AIX environments and kornshell scripting concepts.
  • Continued building comfort with Azure DevOps and Git for version control.

Tools & Technologies

  • SQL DataStage (ETL concepts and workflows)
  • AIX
  • kornshell
  • Azure DevOps
  • Git

Reflection

In this rotation, I learned how enterprise data environments are structured and maintained. Seeing how data flows through different stages and systems helped me understand why standards, governance, and reliability matter so much.

Even without owning a full story, I came away with a better mental model for how data teams operate and how my future software or data work can support them.
