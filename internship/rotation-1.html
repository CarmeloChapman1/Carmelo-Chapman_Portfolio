Rotation I: Cloud Operations & Containers

Supporting repository migrations and automation in a cloud and container-focused environment.

Team & Focus

During this rotation, I worked with the Cloud Operations & Containers team. The team is responsible for maintaining containerized environments and ensuring that deployment pipelines and automation stay reliable as code and tooling evolve.

Key Responsibilities

  • Migrated repositories from GitHub to Azure DevOps as part of a broader platform shift.
  • Updated remotes in Ansible Automation Platform (AAP) so automation jobs used the new ADO repositories.
  • Troubleshot issues related to repository access, credentials, and configuration changes.
  • Documented the migration process so it could be reused and understood by others.

Tools & Technologies

  • Azure DevOps (ADO)
  • Git
  • Ansible / Ansible Automation Platform

Final Product & Reflection

My main final product for this rotation was a GitHub wiki that documented the migration steps and decisions. Creating this documentation helped me practice writing clearly and thinking through the process from another person’s perspective.

Overall, this rotation showed me how important automation, consistency, and documentation are in cloud environments, and it gave me a stronger foundation in version control and deployment practices.
